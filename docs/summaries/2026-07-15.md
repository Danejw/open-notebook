## Key Decisions

- Chat queue is durable (SurrealDB + shared SQLite checkpoints); API and surreal-commands worker must share the same checkpoint file.
- A2UI is opt-in via dual env flags; protocol pinned to A2UI v0.9 / AskUser + Basic catalog over AG-UI, not a second transport.
- Interactive generative UI stays inside assistant message surfaces; deterministic chrome (sidebar, deletes, queue controls) stays non-A2UI.
- Insights are fully removed (not deprecated in place); schema cleaned via migration 39.
- Artifact “clear” action removed from chat UX in favor of simpler active-artifact behavior.
- Working docs folders (checks/critiques/brainstorms/audits/assets/artifacts) deleted after decisions shipped; durable docs remain under numbered docs + `docs/a2ui/`.

## Issues Found

- A2UI and HTML-template interactive patterns may coexist awkwardly until A2UI is proven for more flows.
- Chat queue requires both API and worker processes healthy; idle pending items usually mean the worker is not running.
- Frontend architecture baseline was 64/100 before remediation; follow-up score after UI-013–UI-025 not re-scored in a durable doc after audits folder cleanup.
