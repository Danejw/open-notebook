## Key Decisions

- Native tools are **automatic** for project chat (non-guest); `list_tools` / `get_tool` discover **external MCP tools only**.
- Runtime names use `native__*` prefix; UI/docs/audit use unprefixed names (`get_project_context`).
- Trusted IDs (`project_id` / `session_id`) come only from `CapabilityRuntimeContext`, never from tool arguments.
- `get_skill` / `get_collection` / template gets are **turn-only** (ephemeral); they do not persist session defaults.
- `save_project_artifact` is write-gated and idempotent (`save_idempotency_key`).
- Product language: **project artifacts** (persisted in a project) vs **artifact templates** (global prompt library) vs **output templates** (HTML/structured).
- Surreal table `note` and `note:…` IDs kept for storage compatibility; API/product surface uses project-artifacts.
- Protocol study docs live under `docs/ai-protocols/`; Cos-as-MCP-**server** remains in configuration docs.
